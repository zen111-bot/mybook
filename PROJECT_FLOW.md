# 智能体简历项目开发流程

## 1. 项目目标

打造一个可上线展示的双智能体网页作品，支撑简历展示与面试讲解：

- 个人介绍智能体（Profile Agent）：基于你的个人信息 JSON 进行问答。
- 电商智能体（E-commerce Agent）：提供电商场景下的实用工具能力。
- 网页产品化：用户可在页面中切换智能体并进行交互。

---

## 2. 第一阶段（本周）总路线

本周只做一件事：**跑通最小可用闭环（MVP）**。

闭环定义：

1. 前端页面可输入问题
2. 后端可接收请求
3. LangGraph 可调用对应智能体
4. 智能体能返回可用回答
5. 页面展示回答结果

---

## 3. 执行流程（按顺序）

## Step 1：初始化项目骨架（Day 1）

目标：建立前后端目录和最基本运行能力。

任务：

- 创建目录：`backend/`、`frontend/`、`backend/data/`、`backend/agents/`、`backend/graphs/`
- 后端安装依赖：`fastapi`、`uvicorn`、`langgraph`、`pydantic`
- 创建基础接口：`POST /chat`
- 前端创建聊天页面原型（输入框 + 发送按钮 + 响应区）

验收标准：

- 前端可发请求到后端
- 后端返回固定文本（先不接入智能体）

---

## Step 2：实现个人介绍智能体（Day 2-3）

目标：让系统真正可以“问你是谁”。

任务：

- 新建 `backend/data/profile.json`，录入你的个人信息
- 定义 Profile Agent 的系统提示词（角色、语气、输出风格）
- 用 LangGraph 搭建 Profile graph（路由 -> 读取数据 -> 生成回答）
- 支持常见问答：技能、项目、教育、实习、优势

验收标准：

- 输入“你擅长什么”可返回结构化回答
- 回答内容与 `profile.json` 一致，不胡编

---

## Step 3：实现电商智能体（Day 4-5）

目标：展示你有业务落地能力，不只会聊天。

建议先做两个能力：

1. 商品文案生成（标题 + 卖点）
2. 评论分析（情感倾向 + 改进建议）

任务：

- 定义 E-commerce Agent 的输入格式（商品信息/评论文本）
- 用 LangGraph 搭建 Ecommerce graph（意图识别 -> 对应工具链）
- 前端增加“智能体切换”和“业务输入区域”

验收标准：

- 切换到电商智能体后可正常处理业务输入
- 输出内容可直接用于运营/产品展示

---

## Step 4：联调与体验优化（Day 6）

目标：让它看起来像“作品”，而不是“测试页”。

任务：

- 增加加载状态、错误提示、空状态文案
- 增加示例问题按钮（降低试用门槛）
- 优化回答排版（分点、标题、可读性）

验收标准：

- 非技术用户打开页面也能在 30 秒内上手
- 页面交互完整且无明显报错

---

## Step 5：上线与简历包装（Day 7）

目标：形成可投递可展示的完整作品。

任务：

- 部署前端（建议 Vercel）
- 部署后端（建议 Render/Railway）
- 完善 `README.md`：项目背景、架构、功能、技术栈、运行方式、在线链接
- 准备简历描述（突出“多智能体 + 产品化 + 业务场景”）

验收标准：

- 存在可访问在线地址
- README 清晰，面试官可快速理解项目价值

---

## 4. 每日工作模板（建议）

每天按这个节奏推进：

1. 明确当天目标（只定 1 个主目标）
2. 开发实现（先跑通，再优化）
3. 自测（至少 5 个核心问题）
4. 记录问题与改进点
5. 更新 README（避免最后补文档）

---

## 5. 当前就要做的第一件事

现在立刻开始：

1. 创建项目目录结构
2. 初始化后端 FastAPI
3. 写一个可通的 `POST /chat` 假接口（先返回 `"MVP running"`）

完成这 3 步后，再接入第一个 LangGraph（Profile Agent）。

---

## 6. 里程碑检查点

- M1：前后端通路打通（Day 1）
- M2：Profile Agent 可用（Day 3）
- M3：E-commerce Agent 可用（Day 5）
- M4：Demo 上线可访问（Day 7）

如果任一里程碑延期，优先保留“可演示闭环”，暂时砍掉非核心功能。
